Ebean is an open source "Object Relational Mapping" toolkit for Java. This means it gives you a way to query SQL databases using Java statements and automatically load, save and update database data using Java objects.

The goal of Ebean is to provide a simpler alternative to JPA (Java Persistence Architecture) ORM implementations such as Hibernate and EclipseLink.

JPA was developed to satisfy numerous demands from existing ORM vendors and powerful players in the JEE space with the consequence it is far too complex for todays agile development approach.

Ebean's objectives are to:

  • Provide an ORM toolkit that is fast to learn, code against and maintain.
  • Support developers in their quest to create clean, concise code.
  • Use programming extractions that increase safety while providing maximum performance.
  • Be nothing more than a thin layer over databases and not get in the way of developers.
  • Allow SQL to be a first class citizen when developers want to use it.

Ebean achieves this by providing a clean “sessionless” API and simpler query language.
Relative to JPA, this means:

  • No Session Object (or UnitOfWork or EntityManager).
  • No Attached or Detached Beans.
  • No merge(), persist(), flush(), or clear(). Instead Ebean has save() and delete().
  • You can load objects with raw SQL if you want.

Ebean has been in production for 5 years and is pre-integrated into the popular Play framework.
